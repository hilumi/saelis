# Constellations

Constellations make Saelis's memory **visible, intentional, and entirely user-controlled**. Every
star in the Constellation sky is a memory the user explicitly approved or created — a preference,
a relationship detail, a meaningful event, a helpful context. Nothing appears there silently.

- **Created** by approving a proposal ("Would you like me to remember that?"), optionally edited
  first, or by adding one directly.
- **Placed** deterministically among the Living Sky's star positions (seeded by the memory, never
  by chronology, never by Math.random). The same memory always sits in the same place.
- **Opened** intentionally — private content is never visible until a star is selected, and
  opening a star sends nothing to the model. "Reflect with Saelis" requires a separate, explicit
  choice and passes only the memory's short label through client state (never a query string).
- **Managed** anywhere: edit, reclassify, remove, export — here or in the Memory Center.

More stars are not better. Nothing is counted, scored, or rewarded; a sky with one star and a sky
with forty are equally whole. Empty state: _"Your sky is still open. Memories you choose to keep
will appear here."_
