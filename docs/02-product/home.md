# Home

`/home` — the place a signed-in person arrives. Not a dashboard: no metrics, no task board, no
notifications, no streaks, no cards competing for attention. Home answers _"What would feel
supportive right now?"_ without forcing a choice. Core line: **"Come as you are."** Supporting:
**"There is no rush."**

**Hierarchy.** Living Sky → The Light (visual center; welcoming briefly, then resting; listening
when a door is focused) → time-aware greeting (device-local, hydration-safe, never mood-inferring)
→ four quiet doors (Talk / Arrive / Stay Here / Find one step) → at most ONE dismissible
continuation (conversation > horizon step > recent arrival > North Star) → a Horizon glimpse (one
step, or "The horizon is open.") → a Constellations glimpse (counts only, or "Your sky is still
open.") → a quiet exit ("Step away for now" → one Last Light line, no modal, no blocking).

**Privacy.** Home reads only the user's own presentation-safe data (see home-architecture.md).
Private message text and memory content never appear; continuations carry only user-chosen step
titles. No engagement tracking, no third-party analytics, no location, no weather, no emotional
labels.

**Empty states** are welcoming, never punishing: "Welcome." · "The horizon is open." · "Your sky
is still open." · "Saelis will support you without retained memories."

**Excluded by design:** progress dashboards, overdue labels, red status colors, visit counts,
guilt copy, dependency copy, engagement funnels.

**Dev preview:** `/home?preview=first-visit|returning|horizon|constellations|north-star|empty`
(development only, fictional, never persisted; combines with `?sky=` previews).
